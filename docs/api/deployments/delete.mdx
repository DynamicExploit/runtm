---
title: "Delete Deployment"
api: "DELETE /v0/deployments/{deployment_id}"
description: "Destroy a deployment and release resources"
---

Destroy a deployment, stopping the machine and releasing all associated resources.

## Path Parameters

<ParamField path="deployment_id" type="string" required>
  The deployment ID to destroy (e.g., `dep_abc123xyz`)
</ParamField>

## Response

<ResponseField name="deployment_id" type="string">
  The destroyed deployment ID
</ResponseField>

<ResponseField name="status" type="string">
  Always `destroyed`
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<RequestExample>
```bash cURL
curl -X DELETE "https://app.runtm.com/api/v0/deployments/dep_abc123xyz" \
  -H "Authorization: Bearer runtm_xxx"
```

```python Python
import requests

response = requests.delete(
    "https://app.runtm.com/api/v0/deployments/dep_abc123xyz",
    headers={"Authorization": "Bearer runtm_xxx"}
)

data = response.json()
print(f"Deployment {data['deployment_id']}: {data['message']}")
```

```javascript JavaScript
const response = await fetch(
  "https://app.runtm.com/api/v0/deployments/dep_abc123xyz",
  {
    method: "DELETE",
    headers: {
      "Authorization": "Bearer runtm_xxx"
    }
  }
);

const data = await response.json();
console.log(`Deployment ${data.deployment_id}: ${data.message}`);
```
</RequestExample>

<ResponseExample>
```json 200
{
  "deployment_id": "dep_abc123xyz",
  "status": "destroyed",
  "message": "Deployment destroyed successfully"
}
```

```json 404
{
  "detail": {
    "error": "Deployment not found",
    "deployment_id": "dep_invalid"
  }
}
```

```json 409
{
  "detail": {
    "error": "Cannot destroy deployment in 'building' state",
    "recovery_hint": "Wait for the build/deploy to complete or fail, then try again."
  }
}
```
</ResponseExample>

<Note>
Only deployments in `queued`, `ready`, or `failed` state can be destroyed. Deployments that are currently `building` or `deploying` must complete first.
</Note>
