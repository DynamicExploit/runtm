The Runtm API lets you deploy, manage, and monitor applications programmatically.

## Base URL

```
https://app.runtm.com/api
```

For self-hosted instances, use your own API URL.

## Authentication

All API requests require a Bearer token in the `Authorization` header:

```bash
curl -H "Authorization: Bearer runtm_xxx" https://app.runtm.com/api/v0/deployments
```

Get your API key from [app.runtm.com](https://app.runtm.com) or create one via CLI:

```bash
runtm login
```

## API Scopes

API keys have specific permissions:

| Scope | Permissions |
|-------|-------------|
| `read` | List deployments, view logs, check status |
| `deploy` | Create/update deployments (includes `read`) |
| `delete` | Destroy deployments (includes `read`) |
| `admin` | Full access including token management |

## Rate Limits

| Limit | Value |
|-------|-------|
| Deployments per hour | 10 |
| Max artifact size | 20 MB |
| Build timeout | 10 minutes |
| Deploy timeout | 5 minutes |

## Error Responses

Errors return JSON with an `error` field:

```json
{
  "detail": {
    "error": "Deployment not found",
    "deployment_id": "dep_invalid"
  }
}
```

Common HTTP status codes:

| Code | Meaning |
|------|---------|
| `400` | Invalid request (bad parameters) |
| `401` | Invalid or missing API key |
| `403` | Insufficient permissions |
| `404` | Resource not found |
| `409` | Conflict (e.g., invalid state) |
| `413` | Artifact too large |
| `429` | Rate limit exceeded |
| `500` | Server error |

## Idempotency

For `POST` requests, use the `Idempotency-Key` header to ensure safe retries:

```bash
curl -X POST "https://app.runtm.com/api/v0/deployments" \
  -H "Authorization: Bearer runtm_xxx" \
  -H "Idempotency-Key: unique-request-id-123" \
  -F "manifest=@runtm.yaml" \
  -F "artifact=@artifact.zip"
```

If the same key is used again, the API returns the existing deployment instead of creating a new one.
