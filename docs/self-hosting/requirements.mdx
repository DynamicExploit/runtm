# Requirements

What you need to self-host Runtm.

## System Requirements

### Development/Testing

| Resource | Minimum |
|----------|---------|
| CPU | 2 cores |
| RAM | 4GB |
| Disk | 20GB |
| OS | Linux, macOS, Windows (WSL2) |

### Production

| Resource | Recommended |
|----------|-------------|
| CPU | 4+ cores |
| RAM | 8GB+ |
| Disk | 100GB+ SSD |
| OS | Linux (Ubuntu 22.04+) |

## Software Requirements

### Required

| Software | Version | Purpose |
|----------|---------|---------|
| Docker | 20.10+ | Container runtime |
| Docker Compose | 2.0+ | Service orchestration |
| Git | 2.0+ | Clone repository |

### Optional

| Software | Version | Purpose |
|----------|---------|---------|
| Python | 3.9+ | Local development |
| Node.js | 18+ | Template development |
| flyctl | Latest | Direct Fly.io management |

## Getting a Fly.io Token

### 1. Install flyctl

```bash
# macOS/Linux
curl -L https://fly.io/install.sh | sh

# Windows
powershell -Command "iwr https://fly.io/install.ps1 -useb | iex"
```

### 2. Login

```bash
fly auth login
```

### 3. Create Deploy Token

```bash
fly tokens create deploy -n runtm-worker
```

Save this token - you'll need it in your `.env` file.

## Network Requirements

### Outbound Access

The Runtm services need outbound access to:

| Host | Port | Purpose |
|------|------|---------|
| `api.fly.io` | 443 | Fly.io API |
| `registry.fly.io` | 443 | Docker registry |
| `pypi.org` | 443 | Python packages |
| `registry.npmjs.org` | 443 | NPM packages |

### Inbound Access (Production)

| Port | Service | Purpose |
|------|---------|---------|
| 443 | API | HTTPS API access |
| 80 | API | HTTP redirect to HTTPS |

## Verification

Before proceeding, verify your setup:

```bash
# Check Docker
docker --version
docker compose version

# Check Docker is running
docker info

# Check disk space
df -h

# Check memory
free -h  # Linux
vm_stat  # macOS
```
