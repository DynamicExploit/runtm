# runtm init

Spin up a new sandbox for your agent to build in. Optionally start from a template.

```bash
runtm init [template]
```

## Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `template` | No | Optional template: `backend-service`, `static-site`, or `web-app` |

If not provided, you'll be prompted to choose interactively or start from scratch.

## Options

| Option | Short | Description |
|--------|-------|-------------|
| `--path PATH` | `-p` | Directory to create project in (default: `.`) |
| `--name NAME` | `-n` | Project name (default: directory name) |

## Examples

### Basic Usage

```bash
# Initialize backend service in current directory
runtm init backend-service

# Initialize with project name
runtm init backend-service --name my-api

# Initialize in specific directory
runtm init static-site --path ./my-site
```

### Interactive Mode

```bash
# Prompts you to choose a template
runtm init
```

## What Gets Created

### backend-service

```
my-project/
├── runtm.yaml           # Deployment manifest
├── runtm.discovery.yaml # Search metadata
├── Dockerfile           # Container build
├── pyproject.toml       # Python dependencies
├── app/
│   ├── main.py          # FastAPI entry point
│   ├── api/v1/          # API endpoints
│   ├── services/        # Business logic
│   └── core/            # Config and logging
├── tests/               # Pytest tests
├── README.md            # Documentation
├── AGENT.md             # AI IDE instructions
└── CLAUDE.md            # Claude Code instructions
```

### static-site

```
my-project/
├── runtm.yaml
├── runtm.discovery.yaml
├── Dockerfile
├── package.json
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── globals.css
├── components/
├── public/
│   └── health           # Static health file
├── README.md
├── AGENT.md
└── CLAUDE.md
```

### web-app

```
my-project/
├── runtm.yaml
├── runtm.discovery.yaml
├── Dockerfile
├── frontend/            # Next.js frontend
│   ├── package.json
│   ├── app/
│   └── components/
├── backend/             # FastAPI backend
│   ├── pyproject.toml
│   └── app/
├── README.md
├── AGENT.md
└── CLAUDE.md
```

## After Initialization

Your agent now has an isolated sandbox to work in:

```bash
# Enter the project directory
cd my-project

# Run locally
runtm run

# Deploy when ready
runtm deploy
```

## Customizing Templates

After initialization, customize your project by editing:

1. **`runtm.yaml`** - Add environment variables, features, policy
2. **`runtm.discovery.yaml`** - Add description, tags for searchability
3. **Application code** - Implement your features

See [Templates Overview](/templates/overview) for detailed documentation.
